4.4 Property access expressions

a property access expression evaluates to the value of an object property or an array element.javascript defines two syntaxes for property access.
expression . identifier
expession [expession]

4.4.1 Conditional property access

ES2020 adds two new kinds of property access expressions
expression ?. identifier
expession ?. [expession]
In javascript null and undefined are the only two values which do not have properties.In a regular property access expression using . or [] will throw typeError so use can use ?. syntax to guard agaist these errors.

4.5.1 Conditional invocation

In ES2020, you can also invoke a function using ?.() instead of ().with the new ?.() invocation syntax if the expression to the left of the ?. evaluates to null or undefined then the entire invocation expression evaluates to undefined and no exception is thrown.

4.8.1 The + Operator

The binary + operator adds numeric operands and concatenates string operands. If either of the operand is string or object that converts to a string, the operand is converted to string and concatenation is performed.

4.8.3 Bitwise operators

The bitwise operators perform low level manipulation of the bits in the binary representation of the numbers.The bitwise operators expexct integer operands and behave as if those values are represented in 32 bit integers rather than 64 bit floating point values.These operators convert the operands to numbers if neccessary and coerce the numeric values to 32 bit integers dropping any fractional part.
